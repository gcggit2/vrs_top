# LP Design & Coding Guidelines (Consulting Service Template)

このドキュメントは、`app/consulting/page.tsx` で確立された「高CVRを狙うBtoBサービスLP」のデザイン・実装ルールをまとめたものです。
今後のLP制作・改修時は、本ガイドラインを参照してください。

## 1. デザイン原則 (Design Principles)

### 配色とリズム
- **背景色の交互配置**: `bg-white` と `bg-gray-50` をセクションごとに交互に配置し、視覚的なリズムを作る。
- **アクセントカラー**: `text-brand-red` (`#C50E1D` 想定) を要所で使用。使いすぎず、重要な数字やボタン、強調線に絞る。
- **リッチな装飾**:
  - `bg-gradient-to-b` などの微細なグラデーションで立体感を出す。
  - 背景パターン（ドット、ストライプ、グリッド）を `opacity-10` 程度で薄く敷き、プロっぽさを演出する。

### タイポグラフィ
- **メリハリ**: `text-3xl md:text-5xl` のように、スマホとPCで大胆にサイズを変える。
- **読みやすさ**: 本文は `leading-relaxed` で行間を広めに。
- **強調**: 重要な箇所は `font-black` や `border-b-4` で強調する。

## 2. LP構成テンプレート (Structure)

### 1. First View (FV)
- **ターゲットコール**: 左上にバッジで配置。「○○な方へ」と明記し、自分ごと化させる。
- **キャッチコピー**: 「形容詞」よりも「ベネフィット（どうなるか）」を優先。
- **権威性・信頼**: プロフィール情報（社名・名前・写真）をFV内で見せる。
- **スマホ最適化**: 写真は小さく、テキストを優先。スクロールなしでCTAまで見せる。

### 2. 課題 (Pain)
- ユーザーの悩みを言語化。「そうそう、それで困ってるんだよ」と思わせる。
- チェックリストやアイコンを使って視覚的に。

### 3. 解決策・強み (Solution)
- **比較表**: 競合（大手・格安）との違いを明確にする。○×表ではなく、具体的な言葉で違いを語る。
- **図解**: プロセスや仕組みを簡単な図（Flexboxで並べたアイコン等）で表現する。

### 4. 実績 (Proof)
- 具体的な社名、ロゴ、数字を出す。
- 「なぜ安いのか？」などの疑問に先回りして答える。

### 5. 想い (Emotion)
- 機能的価値だけでなく、代表者のミッションやストーリーを語る。
- 背景色を少し変えるなどして、情緒的なモードに切り替える。

### 6. フロー (Flow)
- 3〜4ステップで簡潔に。「簡単そう」と思わせる。

### 7. CTA (Call to Action)
- **マイクロコピー**: 「入力は1分で完了」「無理な勧誘はしません」など、心理的ハードルを下げる言葉を必ず添える。
- **配置**: FV、中盤、終盤（フッター直前）の最低3箇所に配置。

## 3. コーディング規約 (Coding Rules)

- **Tailwind CSS**: ユーティリティクラスをフル活用する。
- **レスポンシブ**: `md:hidden` (スマホのみ) / `hidden md:block` (PCのみ) を積極的に使い、デザインを妥協しない。
- **アイコン**: 外部ライブラリに依存せず、SVGを直接埋め込む（軽量化とカスタマイズ性のため）。
- **コンポーネント**: `Header`, `Footer` は共通のものを使うが、LPの中身は基本的に `page.tsx` 内にベタ書きでOK（LPは独立性が高いため）。
